# TASK-0009: Export / Import Session

This task adds the ability to save the current session to a JSON file and restore it later. The exported file is a complete, human-readable snapshot that includes the raw occurrence list, all computed statistics, and the current prediction. On import, only the raw occurrences are consumed; statistics and prediction are always recomputed fresh from them.

---

## Goal

1. **Export** — while a session is active, the user can download a `.json` snapshot of the full session state (occurrences + statistics + prediction).
2. **Import** — when no session is active, the user can load a previously exported file; this replaces any current session with the imported occurrences and triggers a full re-render.
3. **Recalculate** — is implicit: because statistics and prediction are always derived live from occurrences, importing an old snapshot and seeing freshly computed numbers *is* the recalculation. No separate action is needed.

---

## New Decision: DEC-0004

Before beginning implementation, append the following entry to `docs/decisions.md`:

```
## DEC-0004 — Export/Import File Format (2026-04-30)

**Decision**: Session snapshots are exported as UTF-8 JSON files with the
following top-level shape:

{
  "version": 1,
  "exportedAt": "<ISO 8601 UTC string>",
  "occurrences": [ "<ISO string>", … ],
  "statistics": <object returned by computeStatistics(), or null>,
  "prediction": <object returned by predictNext(), or null>
}

On **export**: all four fields are populated from live data at the moment of
the download.
On **import**: only `occurrences` is consumed. `statistics` and `prediction`
are ignored; they are always recomputed from the occurrence list.

**Rationale**: Including derived data makes the file a self-contained,
human-readable record useful for sharing or auditing without running the app.
Ignoring it on import guarantees that calculations always reflect the current
engine version, fulfilling the "recalculate" requirement for free.

**Version field**: a `version` integer allows future format migrations. The
current importer must reject any file where `version !== 1` with a descriptive
error.
```

## Steps

### 1. New module: `src/js/sessionIO.js`

Create and export two pure functions. This module has **no DOM dependencies** — it takes and returns plain data, making it fully unit-testable.

#### 1a. `buildExportPayload(occurrences, statistics, prediction)`

```js
/**
 * Assemble the full export payload object.
 *
 * @param {string[]} occurrences   Array of UTC ISO 8601 strings.
 * @param {object|null} statistics Result of computeStatistics(), or null.
 * @param {object|null} prediction Result of predictNext(), or null.
 * @returns {object}               Export payload ready to be JSON-serialised.
 */
export function buildExportPayload(occurrences, statistics, prediction) { … }
```

The returned object must match the shape in DEC-0004 exactly:
- `version`: always `1`.
- `exportedAt`: `new Date().toISOString()` at call time.
- `occurrences`: the array as-is.
- `statistics`: the statistics object (may be `null` for < 2 occurrences).
- `prediction`: the prediction object (may be `null` for < 2 occurrences).

#### 1b. `parseImportPayload(jsonString)`

```js
/**
 * Parse and validate an import JSON string.
 *
 * @param {string} jsonString  Raw file content.
 * @returns {{ ok: true, occurrences: string[] } | { ok: false, error: string }}
 */
export function parseImportPayload(jsonString) { … }
```

Validation rules (in order; stop at first failure):

| Check | Error message to return |
|---|---|
| `jsonString` is not valid JSON | `"Invalid file: not valid JSON."` |
| Parsed value is not a plain object | `"Invalid file: expected a JSON object."` |
| `payload.version` is not `1` | `"Unsupported file version. Only version 1 is supported."` |
| `payload.occurrences` is not an array | `"Invalid file: occurrences must be an array."` |
| `payload.occurrences` is empty | `"Invalid file: no occurrences found."` |
| Any element of `occurrences` is not a string or not a valid ISO date | `"Invalid file: occurrence at index N is not a valid datetime."` |

On success return `{ ok: true, occurrences: payload.occurrences }`.  
On any failure return `{ ok: false, error: '<message>' }`.

**ISO validity check**: `!isNaN(new Date(value).getTime())` is sufficient.

#### 3b. `handleExport()`

```js
function handleExport() {
  const occurrences = loadRecords();
  const statistics  = computeStatistics(occurrences);
  const prediction  = predictNext(occurrences);
  const payload     = buildExportPayload(occurrences, statistics, prediction);

  const blob     = new Blob([JSON.stringify(payload, null, 2)], { type: 'application/json' });
  const url      = URL.createObjectURL(blob);
  const anchor   = document.createElement('a');
  anchor.href     = url;
  anchor.download = `interval-tracker-${new Date().toISOString().slice(0, 10)}.json`;
  anchor.click();
  URL.revokeObjectURL(url);
}
```

Wire it to `#export-session-btn` click inside `initUI()`.

#### 3c. `handleImport(file)`

```js
function handleImport(file) {
  const reader = new FileReader();
  reader.onload = (e) => {
    const result = parseImportPayload(e.target.result);
    if (!result.ok) {
      setImportFeedback(result.error, 'error');
      // Reset the file input so the same file can be re-selected after fixing
      const fileInput = getEl('import-file-input');
      if (fileInput) fileInput.value = '';
      return;
    }

    // Replace session: clear then add imported occurrences as Date objects
    clearRecords();
    addRecords(result.occurrences.map((iso) => new Date(iso)));

    // Clear the file input
    const fileInput = getEl('import-file-input');
    if (fileInput) fileInput.value = '';
    setImportFeedback('', '');

    renderList();
  };
  reader.readAsText(file);
}
```

Wire it to the `change` event of `#import-file-input` inside `initUI()`:

```js
const importInput = getEl('import-file-input');
if (importInput) {
  importInput.addEventListener('change', (e) => {
    if (e.target.files && e.target.files[0]) handleImport(e.target.files[0]);
  });
}
```

#### 3d. `setImportFeedback` / `clearImportFeedback`

Add a dedicated pair of helpers for `#import-feedback`, mirroring the existing pattern:
