# TASK-0006: Statistics Engine

This task implements a pure-logic statistics module (`src/js/statistics.js`) that computes all relevant descriptive and trend statistics about the interval series derived from the persisted occurrences. The results are organised into three progressive levels — **Basic**, **Advanced**, and **Nerd** — each building on the previous. Display layout is out of scope for this task; the module must be fully functional and unit-tested in isolation.

---

## Goal

Given a chronologically sorted array of `Date` objects (or ISO strings), produce a structured statistics object that any future rendering layer can consume. All calculations operate on the **series of intervals** (gaps between consecutive occurrences), not on the raw timestamps themselves, unless stated otherwise.

---

## Definitions

- **Occurrence**: a single recorded event timestamp.
- **Interval**: the time gap between two consecutive occurrences, expressed in **milliseconds** internally. All human-facing values are expressed in the most legible auto-selected unit (see §Unit Formatting).
- **N**: total number of occurrences. The interval series therefore has **N − 1** elements. A minimum of **2 occurrences** is required to produce any statistics; fewer returns `null`.

---

## Unit Formatting

Intervals must be auto-scaled to the most legible unit. The selection rule:

| Median interval | Display unit |
|---|---|
| ≥ 1 day (86 400 000 ms) | days (1 decimal place) |
| ≥ 1 hour (3 600 000 ms) | hours (1 decimal place) |
| otherwise | minutes (0 decimal places) |

All statistics that output an interval value must use the same unit as determined by the median. A `unit` field (`"days"` \| `"hours"` \| `"minutes"`) must be included in the result so the display layer knows how to label values.

## Steps

### 1. Module: `src/js/statistics.js`

Create and export a single top-level function:

```js
/**
 * Compute all statistics for a series of occurrences.
 *
 * @param {string[]|Date[]} occurrences  Chronologically sorted ISO strings or Date objects.
 *                                        Must contain at least 2 entries.
 * @returns {StatisticsResult|null}       null if fewer than 2 occurrences are provided.
 */
export function computeStatistics(occurrences) { … }
```

The returned `StatisticsResult` object has the shape:

```js
{
  unit: 'days' | 'hours' | 'minutes',
  count: number,          // total number of occurrences
  basic: { … },
  advanced: { … },
  nerd: { … },
}
```

---

### 2. Level 1 — Basic

Audience: any user glancing at their data for the first time.

| Field | Description | Formula |
|---|---|---|
| `intervalCount` | Number of intervals (N − 1) | — |
| `mean` | Arithmetic mean of intervals | `Σ intervals / (N−1)` |
| `min` | Shortest interval observed | `Math.min(...intervals)` |
| `max` | Longest interval observed | `Math.max(...intervals)` |
| `range` | Difference between max and min | `max − min` |
| `first` | ISO string of the earliest occurrence | — |
| `last` | ISO string of the most recent occurrence | — |
| `totalSpan` | Total elapsed time from first to last occurrence | `last − first` (ms → scaled unit) |

---

### 3. Level 2 — Advanced

Audience: a user who wants to understand the regularity and shape of their pattern.

| Field | Description | Formula |
|---|---|---|
| `median` | Middle value of the sorted interval series | Standard median (avg of two middle values if N−1 is even) |
| `stdDev` | Population standard deviation of intervals | `√( Σ(xᵢ − μ)² / (N−1) )` |
| `variance` | Population variance | `Σ(xᵢ − μ)² / (N−1)` |
| `cv` | Coefficient of variation (%) — regularity indicator | `(stdDev / mean) × 100` |
| `q1` | First quartile (25th percentile) of intervals | Standard quartile method (nearest-rank) |
| `q3` | Third quartile (75th percentile) of intervals | Standard quartile method (nearest-rank) |
| `iqr` | Interquartile range | `q3 − q1` |
| `trend` | Direction of the interval trend | `"increasing"` \| `"decreasing"` \| `"stable"` — determined by the sign and significance of the linear regression slope (see below) |

**Trend determination**: fit a linear regression `y = a·x + b` where `x` is the interval index (0, 1, 2, …) and `y` is the interval value. If the slope `a` differs from zero by more than 5 % of the mean, classify as `"increasing"` or `"decreasing"`; otherwise `"stable"`.

**Regularity interpretation** (derive from `cv`, include as `regularityLabel`):

| CV | Label |
|---|---|
| < 15 % | `"very regular"` |
| 15 – 35 % | `"regular"` |
| 35 – 75 % | `"irregular"` |
| ≥ 75 % | `"highly irregular"` |

---

### 4. Level 3 — Nerd

Audience: a user who wants the full statistical picture.

| Field | Description | Formula / method |
|---|---|---|
| `mad` | Mean Absolute Deviation | `Σ |xᵢ − μ| / (N−1)` |
| `skewness` | Pearson's moment coefficient of skewness | `( Σ(xᵢ − μ)³ / (N−1) ) / stdDev³` |
| `kurtosis` | Excess kurtosis (Fisher's definition) | `( Σ(xᵢ − μ)⁴ / (N−1) ) / stdDev⁴ − 3` |
| `outliers` | Intervals outside `[Q1 − 1.5·IQR, Q3 + 1.5·IQR]` (Tukey fences) | Array of the actual interval values (scaled unit), empty array if none |
| `outlierCount` | Number of outlier intervals | `outliers.length` |
| `longestStreak` | Longest run of consecutive intervals all within one stdDev of the mean | integer count |
| `regressionSlope` | Raw linear regression slope `a` (in scaled unit per interval index) | from the regression fit used for `trend` |
| `regressionIntercept` | Raw linear regression intercept `b` (in scaled unit) | from the regression fit |
| `r2` | Coefficient of determination of the linear fit | `1 − SS_res / SS_tot` |

---

### 5. Unit Scaling Helper (internal)

Add a private helper:

```js
function msToUnit(ms, unit) {
  if (unit === 'days')    return ms / 86_400_000;
  if (unit === 'hours')   return ms / 3_600_000;
  return ms / 60_000; // minutes
}
```

All output values in the result object must already be scaled (not raw ms), so the display layer never needs to do unit conversion.

---

### 6. Unit Tests (`tests/unit/statistics.test.js`)

All tests must use **deterministic fixed datasets**. Group tests by level and helper.

Required test groups:

- **`computeStatistics` — guard conditions**
  - Returns `null` for empty input.
  - Returns `null` for a single occurrence.
  - Returns a non-null object for exactly 2 occurrences.

- **`computeStatistics` — unit selection**
  - Selects `"minutes"` when median interval < 1 hour.
  - Selects `"hours"` when median interval ≥ 1 hour and < 1 day.
  - Selects `"days"` when median interval ≥ 1 day.

- **Basic level** — verify `mean`, `min`, `max`, `range`, `totalSpan`, `count`, `intervalCount`, `first`, `last` against a known dataset.

- **Advanced level** — verify `median`, `stdDev`, `variance`, `cv`, `q1`, `q3`, `iqr`; verify `trend` for a clearly increasing series, a clearly decreasing series, and a stable series; verify all four `regularityLabel` values.

- **Nerd level** — verify `mad`, `outliers` (non-empty case and empty case), `outlierCount`, `regressionSlope` direction, `r2` for a perfect linear series (should be 1.0 or very close); verify `longestStreak` with a known dataset.

- **Skewness / kurtosis** — verify sign correctness: a right-skewed dataset should produce positive skewness; a normal-ish dataset should produce kurtosis near 0.

Coverage must remain ≥ 80 % branches overall.

### 7. Trivial Display (`src/index.html` + `src/js/uiController.js`)

The goal is to make the computed values visible on the page without designing a final layout. Styling and visual hierarchy are deferred to a future task.

**HTML** — add a `<section id="statistics-section" class="hidden">` below `#occurrences-section`:

```html
<section id="statistics-section" class="hidden">
  <h2>Statistics</h2>
  <div id="statistics-output"></div>
</section>
```

The section is hidden when fewer than 2 occurrences are recorded and shown otherwise.

**`uiController.js`** — add a `renderStatistics()` function that:

1. Calls `computeStatistics(loadRecords())`.
2. If the result is `null`, hides `#statistics-section` and returns.
3. Otherwise, populates `#statistics-output` with three subsections — one `<div>` per level — each containing a `<h3>` label (`"Basic"`, `"Advanced"`, `"Nerd"`) and a flat `<dl>` (description list) of `<dt>` / `<dd>` pairs, one pair per field. Field names can be the raw key names (e.g. `"mean"`, `"stdDev"`); no formatting polish is required.
4. Reveals `#statistics-section`.

`renderStatistics()` must be called from `renderList()` (and from the New Session reset) so the statistics stay in sync with the occurrence list at all times.

No unit tests are required for `renderStatistics()` (DOM-coupled code, excluded per DEC-0001).
