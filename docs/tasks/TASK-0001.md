# TASK-0001: Basic Application Scaffolding

This task establishes the foundational file structure and boilerplate code required by the application architecture described in `docs/architecture.md`. The goal is to create a barebones, functional single-page application (SPA) skeleton.

## Goal:
To create the necessary source directory (`src/`) and structure files/folders (`index.html`, `css/`, `js/`) and implement minimal boilerplate code to prove the correct linking and separation of concerns.

## Steps:

1.  **Directory Creation:** Create the main `src` folder and its subfolders (`src/css` and `src/js`).
2.  **Core Files Creation:**
    *   Create `src/index.html`: This must serve as the root entry point and correctly reference the CSS and JS bundles.
    *   Create `src/css/styles.css`: This file must contain global resets and basic typography to define the application look and feel.
    *   Create `src/js/script.js`: This file must contain the main application initialization logic.
3.  **Boilerplate Implementation:**
    *   **`src/index.html`:** Needs standard HTML boilerplate, linking `styles.css` and `script.js`. A primary content display area placeholder (e.g., `<div id="app-container"></div>`) should be included.
    *   **`src/css/styles.css`:** Must include a CSS reset and define basic global rules (e.g., setting `box-sizing: border-box;`).
    *   **`src/js/script.js`:** Must contain a basic initialization function that executes on page load (e.g., `initializeApp()`) and adds a console log confirmation.

## Checklist of Final State:
*   A new `src/` directory exists.
*   `src/index.html` exists and is linked to `src/css/styles.css` and `src/js/script.js`.
*   `src/css/styles.css` contains basic reset CSS.
*   `src/js/script.js` contains the core initialization loop and logging mechanism.

(Upon completion, subsequent tasks can build upon this solid foundation.)
