# TASK-0011: Complete Application Layout Makeover

---

## Goal

Replace the functional-but-plain layout with a polished, modern UI featuring a light/dark theme, a clear visual hierarchy driven by the prediction output, and a compact active-session workflow. No change to business logic, parser, statistics engine, prediction engine, or data layer.

---

## Design Principles

- **Prediction first.** When a session is active the prediction hero must dominate the viewport.
- **Minimal at rest.** The no-session state is calm and uncluttered — a single onboarding card.
- **Dark + light.** CSS custom properties power both themes. Default follows `prefers-color-scheme`; the user can override and the preference is persisted in `localStorage`.
- **Responsive.** All layouts must work from 320 px upward.
- **No emojis.** Glyphs come from the Lucide icon library (CDN, UMD build, `lucide.createIcons()`).
- **No new business-logic modules.** Only `index.html`, `styles.css`, and `uiController.js` change.

## Steps

### 1. `docs/decisions.md` — DEC-0005

Add an entry logging:
- **Glyph library**: Lucide Icons, loaded via CDN UMD script. Icons are declared as `<i data-lucide="name"></i>` and materialised by calling `lucide.createIcons()` after any DOM mutation that introduces icon placeholders. No build step required.
- **Theme system**: CSS custom properties on `:root` define the light palette; `[data-theme="dark"]` on `<html>` overrides them for the dark palette. On initialisation `uiController.js` reads `localStorage.getItem('theme')`, falls back to `window.matchMedia('(prefers-color-scheme: dark)')`, and sets `data-theme` on `document.documentElement`. A toggle button in the app header persists the choice to `localStorage`.

### 2. `src/index.html` — Restructured markup

Key structural changes:

- Add Google Fonts `<link>` for **Inter** (400, 500, 600, 700).
- Add Lucide CDN `<script>` before the closing `</body>` (after the app module).
- Add `<header class="app-header">` containing the brand title and `<button id="theme-toggle-btn">`.
- **No-session state**: wrap the textarea + submit + import in a `.onboard` card that sits centred on the page.
- **Active-session state**: the `#single-add-section` becomes the unified **ops bar** containing:
  - Left: text input + Add button (the single-add field).
  - Right: New Session, Export Session, Download Report buttons.
  - The `.session-actions` div that used to live inside `#occurrences-details` is removed from there.
- `#occurrences-details` loses the `open` attribute (closed by default).
- Remove the now-empty `.session-actions` from `#occurrences-details`.
- Section order: `#single-add-section`, `#prediction-section`, `#statistics-section`, `#occurrences-section`.

### 3. `src/css/styles.css` — Full rewrite

Sections:

1. **CSS custom properties** — light palette on `:root`, dark palette on `[data-theme="dark"]`.
2. **Reset + base** — box-sizing, Inter font, line-height, color, background from tokens.
3. **App header** — sticky top bar, brand left, theme toggle right. Slim, minimal.
4. **No-session / onboard** — centred card, max-width 560 px, subtle shadow. Large textarea. Primary CTA button. Understated import row below.
5. **Ops bar** — full-width strip below header when session is active. Single-add input grows to fill left side. Action buttons grouped on the right. Wraps on small screens.
6. **Prediction hero** — large surface card. Natural-language tell in a display font size. Date + time pair. Confidence badge (colour-coded: `--success` for High, `--warning` for Medium, `--danger` for Low). Window range with calendar icon. Secondary chips for strategy and interval.
7. **Statistics** — section heading row. Tab bar (Basic / Advanced / Nerd). Active tab has accent underline. Panel: CSS grid of stat tiles (label top, value bottom, value in a larger weight). Only the active panel is visible.
8. **Occurrences** — `<details>` collapsed by default. Custom `<summary>` row with chevron icon. Open state rotates chevron. Rows: slim three-column layout (index | date + time | tell). Alternating row background subtle.
9. **Shared components** — `.chip`, `.confidence-badge`, icon sizing, feedback colours.
10. **Utility** — `.hidden`.
11. **Responsive** — `@media (max-width: 640px)`: ops bar stacks vertically, prediction hero font sizes shrink, stats grid goes to two columns, occurrence rows simplify.

### 4. `src/js/uiController.js` — Updated rendering + theme

#### New / updated helpers

```
humaniseKey(key)  — converts camelCase to Title Case words
fmtVal(v)         — rounds numbers to 2 dp, joins arrays with ", " or "—"
refreshIcons()    — calls lucide.createIcons() if the global exists
confidenceClass(score) — returns 'high' | 'medium' | 'low'

```
