# TASK-0013: Interval Graph View

## Goal

Introduce a minimal, inline-SVG chart that visualises the series of inter-occurrence
intervals so that users can immediately see whether the gaps between events are growing,
shrinking, or staying stable. The chart must be visually coherent with the established
design system (CSS custom properties, light/dark theme, Lucide icons) and must introduce
no external charting library or build-step dependency.

---

## Context & Design Constraints

- **No external libraries.** The chart is rendered as an inline `<svg>` element built
  entirely from `document.createElementNS` calls in a new pure module `chartRenderer.js`.
- **Theme-aware.** All colours are drawn from the CSS custom properties already defined
  on `:root` / `[data-theme="dark"]` and read at render-time via `getComputedStyle`.
  No hard-coded hex values may appear in `chartRenderer.js`.
- **Responsive.** The SVG uses a fixed `viewBox` (e.g. `0 0 600 260`) and
  `preserveAspectRatio="xMidYMid meet"` with `width="100%"` so it scales inside its
  container without JavaScript resize listeners.
- **Placement.** The chart section is inserted between the prediction section and the
  statistics section in `index.html`, making it the first data section the user sees
  after the prediction card.
- **Data source.** Interval values (already converted to the display unit) come from
  `computeStatistics()`. The chart receives the raw intervals array and the `unit` string
  from the statistics result — no additional computation is performed in `chartRenderer.js`
  beyond layout arithmetic.

---

## Visual Specification

### Chart anatomy

```
┌────────────────────────────────────────────────────┐
│  Interval N+1                           [unit]     │  ← section header (reuses existing styles)
│                                                    │
│  Y ┤                                               │
│    │         ●                                     │
│    │    ●    │ ●                                   │
│    │ ●  │    │    ●   ●                            │
│    │ │  │    │    │   │  ●   ●                     │
│    │ │  │    │    │   │  │   │  ●                  │
│    ├─┴──┴────┴────┴───┴──┴───┴──┴──────── X        │
│       1    2    3    4   5   6   7   8             │
│                                                    │
│  ╌╌╌╌  trend   ▲ growing / ▼ shrinking / ─ stable  │
└────────────────────────────────────────────────────┘
```

- **Bars**: each interval is a vertical bar anchored to the X-axis. Bar width is
  uniform; a small gap separates adjacent bars.
- **Dots**: a filled circle sits on top of each bar at the bar's midpoint.
- **Connecting line**: a polyline joins the dot centres, giving a line-chart feel on
  top of the bars.
- **Trend line**: a straight dashed line fitted through the interval series using the
  `regressionSlope` and `regressionIntercept` already present in
  `statistics.nerd`. It spans the full X range.
- **Mean line**: a faint horizontal dotted line at `statistics.basic.mean` for
  reference.
- **Y-axis labels**: 4–5 evenly spaced tick values on the left edge.
- **X-axis labels**: occurrence-gap index (1-based: gap between occurrence 1→2 is
  index 1) shown below each bar.

### Colour coding

The trend line and bar fill adopt a single accent colour that signals the direction of
the interval series:

| Condition | Accent | CSS variable to introduce |
|---|---|---|
| `trend === 'increasing'` (intervals growing) | warm amber/orange | `--color-trend-up` |
| `trend === 'decreasing'` (intervals shrinking) | cool blue | `--color-trend-down` |
| `trend === 'stable'` | neutral (existing `--color-accent`) | — |

Bars use the accent colour at ~30 % opacity; dots and the connecting line use it at
full opacity; the trend line uses it as a dashed stroke. The mean line always uses
`--color-muted` (an existing or new subtle property).

### Minimum data guard

The chart section is hidden (`.hidden`) when fewer than 2 occurrences are stored
(identical to the guard used for statistics).

## Implementation Steps

### Step 1 — CSS custom properties

In `src/css/styles.css`, inside the `:root` block, add:

```css
--color-trend-up:   #e67e22;   /* amber — growing intervals */
--color-trend-down: #2980b9;   /* blue  — shrinking intervals */
--color-muted:      #aaaaaa;   /* neutral reference lines    */
```

Add overrides in `[data-theme="dark"]`:

```css
--color-trend-up:   #f39c12;
--color-trend-down: #3498db;
--color-muted:      #666666;
```

Also add the chart section container styles:

```css
#chart-section { /* same card/section treatment as #prediction-section */ }
.chart-header   { /* label + unit badge, flex row */ }
.chart-wrap     { width: 100%; overflow-x: auto; }
.chart-wrap svg { display: block; width: 100%; }
.chart-legend   { display: flex; gap: 1rem; font-size: 0.75rem; color: var(--color-muted); margin-top: 0.5rem; }
```

### Step 2 — `src/js/chartRenderer.js` (new file)

Create a new ES module that exports a single function:

```js
/**
 * renderIntervalChart(container, intervals, unit, stats)
 *
 * @param {Element}  container  - The DOM element to render the SVG into.
 * @param {number[]} intervals  - Interval values in `unit` (already converted).
 * @param {string}   unit       - 'days' | 'hours' | 'minutes'
 * @param {object}   stats      - Full statistics object from computeStatistics().
 */
export function renderIntervalChart(container, intervals, unit, stats) { … }
```

Internal responsibilities:

1. **Read theme tokens** from `getComputedStyle(document.documentElement)` at call
   time: `--color-accent`, `--color-trend-up`, `--color-trend-down`, `--color-muted`,
   `--color-text`, `--color-border` (or equivalent names already in the stylesheet).

2. **Choose accent colour** based on `stats.advanced.trend`:
   - `'increasing'` → `--color-trend-up`
   - `'decreasing'` → `--color-trend-down`
   - `'stable'`     → `--color-accent`

3. **Layout constants** (internal, not exported):
   - `VIEW_W = 600`, `VIEW_H = 260`
   - `PAD = { top: 20, right: 20, bottom: 40, left: 52 }`
   - Plot area: `plotW = VIEW_W − PAD.left − PAD.right`, `plotH = VIEW_H − PAD.top − PAD.bottom`

4. **Scale helpers**:
   - `xScale(i)` → x pixel for the i-th bar centre (0-indexed)
   - `yScale(v)` → y pixel for value `v`; Y domain is `[0, yMax]` where
     `yMax = max(intervals) × 1.15` (15 % headroom)

5. **Draw in order** (back to front):
   a. Mean reference line (dotted, `--color-muted`, `stroke-width="1"`)
   b. Y-axis ticks + labels
   c. X-axis baseline
   d. Bars (rect, accent at 30 % opacity, `rx="3"`)
   e. Connecting polyline (accent, `stroke-width="2"`, `fill="none"`)
   f. Dots (circle `r="4"`, accent fill)
   g. Trend line (dashed `stroke-dasharray="6 3"`, accent, `stroke-width="1.5"`)
   h. X-axis index labels

6. **Clear** `container.innerHTML` before appending the new SVG.

### Step 3 — `src/index.html`

Add a new section between `#prediction-section` and `#statistics-section`:

```html
<section id="chart-section" class="hidden">
  <div class="chart-header">
    <h2>Interval Graph</h2>
    <span id="chart-unit-badge" class="chip"></span>
  </div>
  <div class="chart-wrap">
    <div id="chart-output"></div>
  </div>
  <div class="chart-legend">
    <span id="chart-trend-label"></span>
  </div>
</section>
```

### Step 4 — `src/js/uiController.js`

1. Add import:
   ```js
   import { renderIntervalChart } from './chartRenderer.js';
   ```

2. Add `renderChart()` function:
   - Load records via `loadRecords()`.
   - Call `computeStatistics(records)`.
   - If result is `null` or fewer than 2 records: add `.hidden` to `#chart-section` and return.
   - Otherwise: remove `.hidden`; extract the raw interval series in ms from the
     records array (consecutive differences), convert to the display unit using the
     same `msToUnit` logic that statistics uses internally — **or** derive them
     directly from `statistics.basic` and `statistics.nerd` data already available.
     
     > **Note**: `computeStatistics` does not currently export the raw interval array.
     > `renderChart()` must compute it locally:
     > ```js
     > const dates = records.map(r => new Date(r));
     > const msIntervals = dates.slice(1).map((d, i) => d - dates[i]);
     > const unitDivisor = unit === 'days' ? 86400000 : unit === 'hours' ? 3600000 : 60000;
     > const intervals = msIntervals.map(ms => +(ms / unitDivisor).toFixed(2));
     > ```
   - Set `#chart-unit-badge` text to the unit string.
   - Set `#chart-trend-label` text to a human-readable direction label:
     - `'increasing'` → `"▲ intervals are growing"`
     - `'decreasing'` → `"▼ intervals are shrinking"`
     - `'stable'`     → `"─ intervals are stable"`
   - Call `renderIntervalChart(document.getElementById('chart-output'), intervals, unit, stats)`.
   - Call `refreshIcons()`.

3. Call `renderChart()` from:
   - `renderList()` (all code paths that currently call `renderStatistics()`)
   - The New Session confirm handler (to clear the chart when the session is reset)

### Step 5 — No unit tests for `chartRenderer.js`

`chartRenderer.js` is a DOM-coupled rendering module (equivalent in status to
`uiController.js`) and is therefore **exempt** from the DEC-0001 unit-test mandate.
No test file is required. Verify the feature manually via `make serve`.

Coverage must remain ≥ 80 % branches across all tested modules after this task.
