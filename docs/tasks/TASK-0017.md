# TASK-0017: Application Favicon

## Goal

Create a polished, brand-consistent SVG favicon for Interval Tracker and wire it into
`src/index.html`. The favicon must:

- Reflect the application's visual identity: the **activity pulse waveform** (the same icon used
  in the app header and chart section) centred on a **rounded-square background**.
- Use the exact accent colour tokens already defined in `src/css/styles.css`:
  - Light mode background: `#4f46e5` (indigo `--accent`)
  - Dark mode background: `#818cf8` (light indigo `--accent` dark value)
  - Waveform stroke: `#ffffff` in both modes
- Support `prefers-color-scheme: dark` natively via an inline `<style>` block inside the SVG,
  so the favicon automatically adapts without JavaScript or a separate file.
- Be a single self-contained `.svg` file that doubles as both the tab favicon and the
  `apple-touch-icon` fallback (via a separate 180 × 180 PNG, see §3 below).
- Render crisply at all favicon sizes: 16 × 16, 32 × 32, 48 × 48, 180 × 180.

---

## Relevant prior decisions

- **DEC-0005** — Lucide `activity` icon is the canonical brand glyph; it is already used in the
  `<header>` and `#chart-section`. The favicon must use the same visual motif to reinforce brand
  recognition.
- No business logic is involved; **DEC-0001 unit-test requirement does not apply** to this task.
- No new JavaScript modules are introduced; no existing modules are modified.

## Section-by-section specification

### 1  SVG favicon file — `src/favicon.svg`

Create a new file `src/favicon.svg` with the following structure:

```
viewBox="0 0 32 32"   (or equivalent square viewport)
```

#### Layout

```
┌─────────────────────────┐
│  rounded-square bg      │  ← rx/ry = 7  (≈ 22 % of 32)
│                         │
│      ╱╲    ╱╲           │
│  ───╱  ╲──╱  ╲──────    │  ← activity waveform, centred
│                         │
└─────────────────────────┘
```

#### Background shape

A `<rect>` covering the full `32 × 32` viewport with `rx="7" ry="7"`.

#### Waveform path

Use the standard Lucide `activity` icon polyline, scaled and translated to fit within
approximately the central **20 × 14** area of the 32 × 32 canvas (i.e. with ~6 px padding on
each side horizontally and ~9 px vertically).

The Lucide `activity` path in its native 24 × 24 coordinate space is:

```
M22 12h-4l-3 9L9 3l-3 9H2
```

Scale this path from 24 × 24 to fit the 20 × 14 target area and translate it to centre it:

- Scale factor x: `20 / 20 ≈ 1.0` (horizontal — the path spans x 2→22, so 20 units)
- Scale factor y: `14 / 18 ≈ 0.78` (vertical — the path spans y 3→21 in native coords; that's
  18 units; target height is 14 px)
- A uniform scale of **`0.78`** applied around the path's own centre, then translated so the
  result sits at approximately `(6, 9)` in the 32 × 32 canvas.

Alternatively, express the waveform as a single `<polyline>` / `<path>` with hand-tuned
coordinates directly in the 32 × 32 space. The exact control points are flexible; the visual
result must be a recognisable heartbeat/EKG waveform centred in the tile.

Suggested hand-tuned path (32 × 32 canvas):

```svg
M2 16 h5 l3.5 -10.5 L16 26 l3.5 -10.5 h5
```

*(x values span 2→27.5, centred; peak/trough span y 5.5→26; adjust to taste.)*

#### Theme-aware colours

Embed a `<style>` block at the top of the SVG `<defs>` (or directly in `<svg>`) to toggle the
background fill between light and dark accent colours:

```svg
<style>
  .bg  { fill: #4f46e5; }
  .fg  { stroke: #ffffff; }
  @media (prefers-color-scheme: dark) {
    .bg { fill: #818cf8; }
  }
</style>
```

Apply `class="bg"` to the `<rect>` and `class="fg"` to the waveform `<path>` or `<polyline>`.
The `<path>`/`<polyline>` must have `fill="none"` and an appropriate `stroke-width` (suggested:
`stroke-width="2.5"` at 32 × 32) with `stroke-linecap="round"` and `stroke-linejoin="round"`.

#### Full SVG template

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <style>
    .bg { fill: #4f46e5; }
    .fg { fill: none; stroke: #ffffff; stroke-width: 2.5;
          stroke-linecap: round; stroke-linejoin: round; }
    @media (prefers-color-scheme: dark) {
      .bg { fill: #818cf8; }
    }
  </style>
  <rect class="bg" width="32" height="32" rx="7" ry="7"/>
  <path class="fg" d="M2 16 h5 l3.5 -10.5 L16 26 l3.5 -10.5 h5"/>
</svg>
```

> The implementer must visually verify the result in a browser at 16 × 16 and 32 × 32 effective
> sizes and adjust the path coordinates or stroke-width if needed. The above values are a
> strongly recommended starting point, not a hard constraint.

### 2  Wiring in `src/index.html`

Add the following `<link>` tags inside `<head>`, after `<meta charset>` and before the font
`<link rel="preconnect">` tags:

```html
<!-- Favicon -->
<link rel="icon" type="image/svg+xml" href="favicon.svg" />
<link rel="icon" type="image/png" sizes="32x32" href="favicon-32.png" />
<link rel="apple-touch-icon" sizes="180x180" href="apple-touch-icon.png" />
```

The SVG `<link>` must be the first `rel="icon"` entry so modern browsers pick it up. The PNG
fallbacks are declared for completeness but their physical files are optional — if they are not
generated, the `<link>` tags for them may be omitted.

---

### 3  PNG fallback files (optional but recommended)

If an SVG-to-PNG conversion tool is available (e.g. Inkscape, `rsvg-convert`, `sharp`, or an
online converter), generate:

- `src/favicon-32.png` — 32 × 32 px, from `favicon.svg` at light-mode colours.
- `src/apple-touch-icon.png` — 180 × 180 px, from `favicon.svg` at light-mode colours, with a
  slightly thicker stroke (≈ 3.5 px scaled proportionally).

If no conversion tooling is available, omit the PNG files and only wire the SVG `<link>`.

---

## Implementation steps

1. Create `src/favicon.svg` following the template in §1. Open the file in a browser tab to
   visually verify it renders correctly before wiring it.
2. Add the `<link rel="icon" type="image/svg+xml" href="favicon.svg" />` tag to `<head>` in
   `src/index.html`.
3. If PNG fallbacks are generated, place them in `src/` and add their `<link>` tags too.
4. Serve the app with `make serve` and confirm the favicon appears in the browser tab.
5. Update `docs/progress.md` with a TASK-0017 entry.
