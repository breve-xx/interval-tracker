# TASK-0004: Makefile for Common Development Operations

This task introduces a `Makefile` at the project root that exposes a consistent, memorable interface for all routine development operations. All commands documented in `docs/testing.md` and any future operational needs must be accessible through `make` targets.

---

## Goal

Provide a single, self-documenting `Makefile` so that any developer (or AI agent) can discover and execute common operations without needing to memorise npm script names or remember tooling flags.

---

## Steps

### 1. Prerequisites Check

The Makefile must guard the `serve` target against missing dependencies. If `node_modules/` does not exist, it must run `npm install` automatically before proceeding.

### 2. Makefile Targets

Create a `Makefile` at the project root with at least the following targets:

| Target | Description |
|---|---|
| `help` | Print a formatted list of all available targets with short descriptions. Must be the **default** target (i.e. running bare `make` shows help). |
| `install` | Install all npm dependencies (`npm install`). |
| `serve` | Start a local development server with live-reloading. Depends on `node_modules/` being present. Use **Vite**'s dev server (`npx vite src/`) since Vite is already a project dependency (pulled in by Vitest). The server should open the browser automatically. |
| `test` | Run the full test suite once (`npm test`). |
| `test-watch` | Run tests in interactive watch mode (`npm run test:watch`). |
| `coverage` | Run tests with coverage reporting (`npm run test:coverage`). |
| `clean` | Remove `node_modules/` and any generated artefacts (e.g. `coverage/`). |

### 3. `help` Target Implementation

Use a self-documenting convention: lines of the form `## <description>` directly above each target are extracted and printed. Example output:

```
Usage: make <target>

  help          Show this help message
  install       Install npm dependencies
  serve         Start live-reloading dev server
  test          Run unit tests once
  test-watch    Run unit tests in watch mode
  coverage      Run tests with coverage report
  clean         Remove node_modules and generated files
```

### 4. npm Script Alignment

Ensure `package.json` includes a `"dev"` script that mirrors the `serve` target (`"vite src/"`) so the Makefile delegates cleanly:

```json
"scripts": {
  "dev":           "vite src/",
  "test":          "vitest run",
  "test:watch":    "vitest",
  "test:coverage": "vitest run --coverage"
}
```

If `package.json` does not yet exist (it will after TASK-0003), this task must create it.

### 5. Vite Dependency

Add `vite` as a dev dependency if not already present:

```bash
npm install --save-dev vite
```

Vite serves the `src/` directory directly without a build step, matching the front-end-only architecture defined in `docs/architecture.md`.

---

## Checklist of Final State

- `Makefile` exists at the project root.
- Running bare `make` prints the help message.
- `make install` runs `npm install` successfully.
- `make serve` starts a Vite dev server pointing at `src/`, opening the browser and hot-reloading on file changes.
- `make test` runs the Vitest suite and exits 0 when all tests pass.
- `make test-watch` launches Vitest in watch mode.
- `make coverage` produces a coverage report.
- `make clean` removes `node_modules/` and `coverage/`.
- `package.json` contains the `dev` npm script.

(Upon completion, all routine operations are accessible via a single, discoverable interface.)
