# TASK-0005: Recorded Occurrences Display Section

This task implements a minimal, read-only display section for all persisted occurrences, along with a "New Session" control that clears all stored data so the user can start a fresh import.

---

## Goal

Provide a clear, formatted view of every recorded occurrence so the user can verify what has been imported. Occurrences cannot be edited or deleted individually; the only mutation operation available is a full reset via "New Session".

---

## Steps

### 1. HTML Structure (`src/index.html`)

Add a `<section id="occurrences-section">` below the input section containing:

- A `<h2>` heading (e.g. "Recorded Occurrences").
- An `<ol id="occurrence-list">` ordered list where each `<li>` is a self-contained card (see ┬¦2).
- A `<button id="new-session-btn">New Session</button>` placed at the bottom of the section.

The section should be hidden (e.g. via a `.hidden` utility class) when there are no recorded occurrences and visible otherwise.

### 2. Card Layout per Occurrence (`src/index.html` + `src/js/uiController.js`)

Each `<li>` in `#occurrence-list` is rendered as a card. The card contains **two layers**:

**Primary layer** ŌĆö the main visual content of the card:

| Element | Format | Example |
|---|---|---|
| Date | `dd/MM/yyyy` | `15/03/2024` |
| Time | `HH:mm` (24 h) | `14:30` |

Date and time are displayed as the dominant text inside the card (large or bold), side by side or stacked ŌĆö implementer's choice, as long as they are visually prominent.

**Accessory layer** ŌĆö a secondary, visually subdued line beneath the primary content:

- Pattern: `The {ordinal day} of {full month name} at {HH:mm}`
- Example: `The 1st of January at 11:55`, `The 22nd of March at 08:04`
- Rendered in a smaller font size and/or muted colour to signal it is supplementary.

Suggested card markup:

```html
<li class="occurrence-card">
  <div class="occurrence-card__primary">
    <span class="occurrence-card__date">15/03/2024</span>
    <span class="occurrence-card__time">14:30</span>
  </div>
  <p class="occurrence-card__tell">The 15th of March at 14:30</p>
</li>
```

Both the primary and accessory values must be derived from the stored `Date` object, regardless of original input format.

Helper functions to extract and unit-test:

- `formatOccurrenceDate(date)` ŌåÆ `"dd/MM/yyyy"` string
- `formatOccurrenceTime(date)` ŌåÆ `"HH:mm"` string
- `formatOccurrenceTell(date)` ŌåÆ `"The Nth of Month at HH:mm"` string

Ordinal suffixes follow standard English rules:

| Suffix | Days |
|---|---|
| `st` | 1, 21, 31 |
| `nd` | 2, 22 |
| `rd` | 3, 23 |
| `th` | everything else (including 11, 12, 13) |

### 4. New Session Button (`src/js/uiController.js` + `src/js/dataService.js`)

- The "New Session" button is rendered inside `#occurrences-section`.
- Clicking it must:
  1. Display a simple inline confirmation prompt (e.g. change the button text to "Confirm ŌĆö this will erase all data" and require a second click to proceed), **or** use a native `window.confirm` dialog ŌĆö implementer's choice.
  2. On confirmation: call `dataService.clearRecords()` (new function, see ┬¦5), clear the occurrences list, hide `#occurrences-section`, and reset the input area to its initial state (empty textarea, no feedback message).
- On cancellation (second-click guard or `window.confirm` cancel): do nothing.

### 5. `dataService.clearRecords()` (`src/js/dataService.js`)

Add and export a new function:

```js
/**
 * Erase all persisted records from localStorage.
 */
export function clearRecords() {
  localStorage.removeItem(STORAGE_KEY);
}
```

This function must be covered by a unit test in `tests/unit/dataService.test.js`.
