# TASK-0010: Markdown Report Download

This task adds a "Download Report" feature that generates a well-formatted Markdown file from the current session data (occurrences, statistics, prediction) and triggers a browser download. The report uses a rudimentary template defined in this task; the final visual design of the template is deferred to a future task.

---

## Goal

While a session is active, the user can download a `.md` file that presents the same information contained in the JSON export in a human-readable, structured Markdown document. The file is generated entirely client-side — no backend, no new dependencies.

#### Rudimentary template structure

The generated document must follow this section order:

```
# Interval Tracker — Session Report

Generated: <exportedAt formatted as a human-readable local datetime>
Occurrences: <count>

---

## Occurrences

| # | Date | Time |
|---|------|------|
| 1 | DD/MM/YYYY | HH:MM |
| … | … | … |

---

## Statistics

*(omit entire section if statistics is null)*

**Unit:** <unit>

### Basic
| Metric | Value |
|--------|-------|
| <key>  | <value> |
…

### Advanced
| Metric | Value |
…

### Nerd
| Metric | Value |
…

---

## Prediction

*(omit entire section if prediction is null)*

| Field            | Value         |
|------------------|---------------|
| Predicted date   | DD/MM/YYYY HH:MM |
| Window earliest  | DD/MM/YYYY HH:MM |
| Window latest    | DD/MM/YYYY HH:MM |
| Confidence       | <score>% — <label> |
| Strategy         | <strategy>    |
| Interval used    | <N> minutes   |
```

**Formatting rules:**

- **Generated timestamp**: use `new Date().toLocaleString()` (locale-default, no dependency on `formatters.js` — this keeps `sessionIO.js` dependency-free).
- **Occurrences table**: each row uses the ISO string split at `T`; date part formatted as `DD/MM/YYYY` (parse the ISO string directly — do not import `formatters.js`).
- **Statistics tables**: iterate `Object.entries()` on `basic`, `advanced`, and `nerd`. Array values (e.g. `outliers`) rendered as a comma-separated list or `—` if empty. Numeric values rendered as-is (already scaled by `statistics.js`).
- **Prediction table**: `intervalUsedMs` converted to minutes via `Math.round(value / 60_000)`.
- Sections that depend on null data (`statistics`, `prediction`) are **omitted entirely** from the output, not rendered as empty tables.

### 3. `src/js/uiController.js` — wiring

#### 3a. Import `buildMarkdownReport`

```js
import { buildExportPayload, parseImportPayload, buildMarkdownReport } from './sessionIO.js';
```

#### 3b. `handleDownloadReport()`

```js
function handleDownloadReport() {
  const occurrences = loadRecords();
  const statistics  = computeStatistics(occurrences);
  const prediction  = predictNext(occurrences);
  const markdown    = buildMarkdownReport(occurrences, statistics, prediction);

  const blob   = new Blob([markdown], { type: 'text/markdown' });
  const url    = URL.createObjectURL(blob);
  const anchor = document.createElement('a');
  anchor.href     = url;
  anchor.download = `interval-tracker-report-${new Date().toISOString().slice(0, 10)}.md`;
  anchor.click();
  URL.revokeObjectURL(url);
}
```

Wire to `#download-report-btn` click inside `initUI()`.
