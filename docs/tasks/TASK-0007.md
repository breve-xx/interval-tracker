# TASK-0007: Next Occurrence Prediction Engine

This task implements a pure-logic prediction module (`src/js/prediction.js`) that estimates the next occurrence of the tracked event based on the statistical profile of the interval series. The result includes a predicted datetime, a numeric confidence score, a human-readable confidence label, and an earliest/latest window bounding the prediction.

---

## Goal

Given a chronologically sorted array of occurrences, select the most appropriate prediction strategy based on the statistical characteristics of the data, compute the predicted next occurrence, and quantify how much trust to place in that prediction.

---

## Definitions

- **Predicted interval**: the estimated gap (in ms) between the last known occurrence and the next one.
- **Predicted date**: `last occurrence + predicted interval`.
- **Confidence score**: an integer in `[0, 100]` representing how reliably the model can predict the next occurrence. 100 = certain, 0 = no basis for prediction.
- **Confidence window**: `[earliestDate, latestDate]` — a ±1 stdDev interval around the predicted date, representing the range in which the next occurrence is most likely to fall.

---

### 1. Module: `src/js/prediction.js`

```js
/**

 *                                        Must contain at least 2 entries.

export function predictNext(occurrences) { … }
```

The returned `PredictionResult` object has the shape:

```js
{
  predictedDate    : string,   // ISO 8601 string of the predicted next occurrence
  earliestDate     : string,   // ISO 8601 string — lower bound (predicted − 1×stdDev)
  latestDate       : string,   // ISO 8601 string — upper bound (predicted + 1×stdDev)
  confidence       : number,   // integer 0–100
  confidenceLabel  : string,   // 'high' | 'moderate' | 'low'
  strategy         : string,   // which strategy was applied (see §2)
  intervalUsedMs   : number,   // raw predicted interval in milliseconds
}
```

The module must import and call `computeStatistics` from `./statistics.js` internally. All strategy decisions are derived exclusively from the statistics object — `prediction.js` must not re-implement any statistical calculation.

---

### 2. Strategy Selection

The prediction strategy is chosen by inspecting `advanced.trend` and `advanced.regularityLabel` from the statistics result:

| Condition | Strategy | Predicted interval |
|---|---|---|
| `trend === 'stable'` and `regularityLabel` is `'very regular'` or `'regular'` | `'mean'` | `basic.mean` (converted back to ms) |
| `trend === 'stable'` and `regularityLabel` is `'irregular'` or `'highly irregular'` | `'median'` | `advanced.median` (converted back to ms) |
| `trend === 'increasing'` or `trend === 'decreasing'` | `'regression'` | extrapolated via the regression model: `(nerd.regressionSlope × N_intervals + nerd.regressionIntercept)` converted back to ms, clamped to a minimum of 1 minute so the prediction is never in the past |

Where `N_intervals` is the number of intervals already observed (`basic.intervalCount`), i.e. the index of the **next** interval to be predicted.

**Unit reconversion**: `statistics.js` outputs values already scaled to the display unit. To convert back to ms:

```js
function unitToMs(value, unit) {
  if (unit === 'days')  return value * 86_400_000;
  if (unit === 'hours') return value * 3_600_000;
  return value * 60_000; // minutes
}
```

---

### 3. Confidence Score

The confidence score is an integer in `[0, 100]` computed as follows:

```
base        = max(0, 100 − cv)          // CV=0 → 100, CV=100 → 0, CV>100 → 0
sampleBonus = min(10, (count − 2) × 2)  // +2 per occurrence above 2, capped at +10
outlierPenalty = (outlierCount / intervalCount) × 30  // up to −30 for all-outlier series
confidence  = clamp(round(base + sampleBonus − outlierPenalty), 0, 100)
```

Where `cv`, `count`, `outlierCount`, and `intervalCount` all come from the statistics object.

**Confidence label** derived from the score:

| Score | Label |
|---|---|
| ≥ 75 | `'high'` |
| 40 – 74 | `'moderate'` |
| < 40 | `'low'` |

---

### 4. Confidence Window

```
stdDevMs     = unitToMs(advanced.stdDev, unit)
earliestDate = new Date(lastTimestamp + intervalUsedMs − stdDevMs)
latestDate   = new Date(lastTimestamp + intervalUsedMs + stdDevMs)
```

If `earliestDate` would fall before or at the last known occurrence (which can happen when `stdDevMs > intervalUsedMs`), clamp it to `lastTimestamp + 1 ms` so the window always lies in the future relative to the last occurrence.

### 5. Unit Tests (`tests/unit/prediction.test.js`)

All tests must use **deterministic fixed datasets** consistent with those used in `statistics.test.js`.

Required test groups:

- **Guard conditions**
  - Returns `null` for empty input.
  - Returns `null` for a single occurrence.
  - Returns a non-null result for exactly 2 occurrences.

- **Strategy selection**
  - Uses `'mean'` strategy for a very regular stable series (uniform gaps).
  - Uses `'median'` strategy for a highly irregular stable series.
  - Uses `'regression'` strategy for a clearly increasing series.
  - Uses `'regression'` strategy for a clearly decreasing series.

- **Predicted date**
  - `predictedDate` is after the last occurrence in all cases.
  - For a perfectly uniform series the predicted date equals `last + mean interval` (within 1 ms rounding).
  - For a regression-based prediction the predicted date lies further from the last occurrence than the mean for an increasing series (intervals getting longer).

- **Confidence score**
  - Score is in `[0, 100]` for all tested datasets.
  - A perfectly uniform series scores higher than a highly irregular one.
  - Adding more occurrences to a stable series increases or maintains the score (sampleBonus effect).

- **Confidence label**
  - Returns `'high'` for a uniform series with enough occurrences.
  - Returns `'low'` for a highly irregular series with few occurrences.

- **Confidence window**
  - `earliestDate` ≤ `predictedDate` ≤ `latestDate` (ISO string comparison).
  - `earliestDate` is strictly after the last occurrence.
  - For a perfectly uniform series (stdDev = 0), `earliestDate === predictedDate === latestDate`.

- **Regression interval clamping**
  - A decreasing series that extrapolates to a negative or near-zero interval still produces a `predictedDate` after the last occurrence.

Coverage must remain ≥ 80 % branches overall.
