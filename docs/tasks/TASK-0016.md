# TASK-0016: Redesigned Markdown Report

## Goal

Replace the current `buildMarkdownReport` implementation in `src/js/sessionIO.js` with a rich,
readable Markdown document that closely mirrors the information hierarchy and content of the live
application page — while staying within what Markdown can render cleanly.

The current report is a collection of raw data tables. The new report must:

- Use a **hardcoded template** (a literal multi-line string with named slots) filled with live data.
- Replace raw `| Metric | Value |` tables with **definition-style blocks** (bold label → value →
  italic description on consecutive lines), giving each figure a plain-English explanation so the
  report is self-contained and readable without the app.
- Port the interval chart as an **ASCII bar chart** built from Unicode block characters.
- Make the prediction section **time-aware** (show how far away the prediction is from *now*) and
  include a plain-English confidence narrative.
- Keep the occurrences list **discrete**: a compact numbered list, not a full table.
- Update all dependent tests to cover the new output.

---

## Relevant prior decisions

- **DEC-0001** — unit tests required for business logic.
- **DEC-0004** — `buildMarkdownReport` is a pure function in `sessionIO.js`; no DOM or
  localStorage dependencies allowed.
- `docs/testing.md` — Vitest + jsdom, ≥ 80 % branches.

## Section-by-section specification

### 1  Header

```
# Interval Tracker — Session Report

Generated: {date} at {time}   ·   {count} occurrences recorded
```

`{date}` and `{time}` are the local date/time at generation, formatted as `DD/MM/YYYY` and `HH:MM`.
`{count}` is `occurrences.length`.

---

### 2  Prediction (shown first — most actionable, mirrors the hero position on the page)

Shown only when `prediction !== null`.

```
---

## ▶ Next predicted occurrence

**{tell}**
{date}  ·  {time}

**Confidence: {score}% — {label}**
_{confidence narrative}_

**Prediction window:** {earliest date} {earliest time} → {latest date} {latest time}
_Likely range: ± one standard deviation around the predicted date._

**Strategy:** {strategy}
_{strategy description}_

**Interval used:** {interval used, in the session unit, rounded to 2 dp} {unit}
```

`{tell}` is the human-readable tell string (e.g. "The 14th of June at 09:30") derived from
`predictedDate` using the same `formatOccurrenceTell` logic already present in `formatters.js`.

`{confidence narrative}` must match the same text used in `renderPrediction()` in `uiController.js`:
- score ≥ 70: `"High confidence (≥70 %) — historical intervals are very consistent, so the estimate is likely accurate."`
- score 40–69: `"Moderate confidence (40–69 %) — some variability in your intervals; treat the window as a rough guide."`
- score < 40: `"Low confidence (<40 %) — intervals are irregular, so the prediction may be off by a significant margin."`

`{strategy description}` must match `STRATEGY_DESC` in `uiController.js`:
- `mean`: `"Projects from the last occurrence using the average gap. Best for very consistent patterns."`
- `median`: `"Projects using the median gap — more robust when a few unusual intervals exist."`
- `regression`: `"Fits a trend line to the interval series and extrapolates. Used when gaps are clearly growing or shrinking."`

**Time-to-prediction block** (shown only when the predicted date is in the future):

```
**Time remaining:** {days} d  {hours} h  {minutes} min
```

Computed as `predictedDate − new Date()`. When the predicted date is in the past, show:

```
**Note:** The predicted date has already passed.
```

### 3  Interval chart

Shown only when `statistics !== null` (requires at least 2 occurrences, i.e. at least one interval).

```
---

## Interval Chart  [{unit}]

{ascii bar chart}

Trend: {trend arrow + label}   ·   Mean: {mean, rounded to 2 dp} {unit}
```

#### ASCII bar chart construction

1. Build the raw interval array from consecutive occurrence differences (in `{unit}`), identical to
   the array passed to `renderIntervalChart` in `uiController.js`.
2. Normalise each interval to a bar height in the range `[0, 7]`:
   `barIndex = Math.round((value / maxValue) * 7)`, clamped to `[0, 7]`.
   Use the Unicode block characters `[' ', '▁', '▂', '▃', '▄', '▅', '▆', '▇', '█']` (index 0–8;
   a zero-height bar uses the space character, index 0 maps to `▁` for a minimal-height bar — so
   `barIndex` maps to `blocks[barIndex + 1]`).
   Wait — revise: map `barIndex` 0 → `▁` (smallest visible), 7 → `█` (full). So use
   `blocks = ['▁','▂','▃','▄','▅','▆','▇','█']` and `blocks[Math.min(7, barIndex)]`.
3. Render each bar as a column of characters followed by a space, all on a single line. Place the
   chart inside a fenced code block so spacing is preserved in all Markdown renderers:
   ````
   ```
   ▁ ▃ █ ▅ ▂ ▇ ▄
   ```
   ````
4. Below the code block: `Gap #1  Gap #2  …  Gap #N` as a compact label line (only when ≤ 12
   intervals; omit for dense series). This is a plain text line, **not** inside the code block.
5. Trend arrow: `▲ intervals are growing` / `▼ intervals are shrinking` / `─ intervals are stable`.

### 4  Statistics

Shown only when `statistics !== null`.

```
---

## Statistics  [{unit}  ·  {count} occurrences]

### Basic
_{STATS_LEVEL_DESC.basic}_

**Interval Count:** {value}
_The number of gaps between consecutive occurrences._

**Mean:** {value} {unit}
_The average gap length — a typical interval between events._

...one block per key in `statistics.basic` in the same order as the object...

### Advanced
_{STATS_LEVEL_DESC.advanced}_

...one block per key in `statistics.advanced`...

### Nerd
_{STATS_LEVEL_DESC.nerd}_

...one block per key in `statistics.nerd`...
```

#### Definition-block format (per metric)

Each metric is rendered as exactly three lines:

```
**{Human-readable label}:** {formatted value} [{unit, only for numeric interval values}]
_{plain-English description from STATS_GLOSSARY}_

```

(Followed by a blank line to separate from the next metric.)

Formatting rules:
- Numbers rounded to 2 decimal places (same as `round2` in `uiController.js`).
- Arrays (`outliers`): comma-joined numbers (each rounded to 2 dp), or `—` when empty.
- ISO datetime strings (`first`, `last`): formatted as `DD/MM/YYYY HH:MM`.
- String values (`regularityLabel`, `trend`): as-is.
- Unit suffix is appended only to numeric values that represent interval lengths:
  `mean`, `min`, `max`, `range`, `totalSpan`, `median`, `stdDev`, `variance`, `q1`, `q3`, `iqr`,
  `mad`, `outliers` (each element), `regressionSlope`, `regressionIntercept`.
  (`cv`, `r2`, `skewness`, `kurtosis`, `outlierCount`, `longestStreak`, `intervalCount` do **not**
  get a unit suffix.)

The `STATS_GLOSSARY` and `STATS_LEVEL_DESC` constants must be **moved from `uiController.js` into
`sessionIO.js`** (or into a new shared module) so that `buildMarkdownReport` can reference them
without creating a circular dependency. `uiController.js` then imports them from the same source.

> **Decision to record**: A new architectural decision DEC-0006 must be added to `docs/decisions.md`
> documenting that `STATS_GLOSSARY`, `STATS_LEVEL_DESC`, and `STRATEGY_DESC` are canonical shared
> constants living in `src/js/reportConstants.js`, importable by both `sessionIO.js` and
> `uiController.js`.

### 5  Occurrences (discrete — kept compact at the end)

```

## Occurrences ({count})

1. DD/MM/YYYY HH:MM
2. DD/MM/YYYY HH:MM
...
```

A plain numbered list. One line per occurrence: date and time. No additional columns, no table.
For sessions with more than 50 occurrences, show only the first 10 and last 10, with a separator:

```
1.  DD/MM/YYYY HH:MM
2.  DD/MM/YYYY HH:MM
...
10. DD/MM/YYYY HH:MM

… {count − 20} more occurrences …

{count − 9}. DD/MM/YYYY HH:MM
...
{count}. DD/MM/YYYY HH:MM
```

4. Add `DEC-0006` to `docs/decisions.md`.

### Step 2 — Implement helper functions in `sessionIO.js`

Add or rewrite the following private helpers:

- `isoToDate(iso)` — unchanged (already exists).
- `isoToTime(iso)` — unchanged (already exists).
- `isoToTell(iso)` — reproduce the `formatOccurrenceTell` logic inline (or import from

- `buildChartSection(statistics, occurrences)` — returns the full §3 block.
