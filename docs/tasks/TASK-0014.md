# TASK-0014: Application Footer

## Goal

Add a minimal, tasteful footer to the application that anchors the page, reinforces the
product identity, and links to the GitHub repository. The footer must be visually coherent
with the established design system and must not introduce any new dependencies.

---

## Context & Design Constraints

- **Placement**: the footer is a direct child of `#app-container`, after all content sections.
- **Always visible**: the footer is rendered unconditionally — it is not gated behind session
  state and never carries the `.hidden` class.
- **No new JavaScript**: the footer is static HTML. No module changes are required beyond
  adding the markup to `index.html` and the styles to `styles.css`.
- **GitHub icon**: the GitHub logo precedes the repository link as an **inline SVG `<path>`**
  directly in the HTML. Lucide 1.14.0 (the version resolved by the CDN `@latest` tag) does
  not include brand icons — `data-lucide="github"` is silently ignored by `createIcons()`.
  The inline SVG uses `class="app-footer__github-icon"` sized at `1em × 1em` with
  `fill: currentColor` so it inherits the link colour and theme transitions for free.
- **Tone**: subdued and minimal — the footer should recede into the background rather than
  compete with the content above it.

---

## Visual Specification

```
────────────────────────────────────────────
  Interval Tracker · Open source   ⌘ GitHub
────────────────────────────────────────────
```

- A single `1px` top border using `var(--border)` separates the footer from the content.
- Left side: the app name in muted weight + a brief tagline ("Open source interval analysis").
- Right side: a GitHub icon + link reading "View on GitHub", opening in a new tab.
- Both sides use `var(--text-subtle)` as the base colour; the link transitions to
  `var(--text-muted)` on hover, with no underline at rest.
- The footer has generous top padding so it breathes below the last content section.
- On narrow viewports (`max-width: 640px`) the two sides stack vertically, centred.

---

## Implementation Steps

### Step 1 — `src/index.html`

Add the following element as the last child of `#app-container`, after
`#occurrences-section`:

```html
<footer class="app-footer">
  <span class="app-footer__brand">
    Interval Tracker &mdash; open source interval analysis
  </span>
  <a
    class="app-footer__link"
    href="https://github.com/breve-xx/interval-tracker"
    target="_blank"
    rel="noopener noreferrer"
    aria-label="View Interval Tracker on GitHub"
  >
    <i data-lucide="github"></i>
    View on GitHub
  </a>
</footer>
```

### Step 2 — `src/css/styles.css`

Add footer styles inside the existing `/* ── Responsive */` section boundary —
i.e., **before** the `@media` block — so the responsive override can reference
`.app-footer` cleanly:

```css
/* ── Footer ────────────────────────────────────────────────────────────────── */

.app-footer {
  display         : flex;
  align-items     : center;
  justify-content : space-between;
  gap             : 1rem;
  padding         : 2rem 0 1rem;
  margin-top      : 2rem;
  border-top      : 1px solid var(--border);
}

.app-footer__brand {
  font-size  : 0.8rem;
  color      : var(--text-subtle);
}

.app-footer__link {
  display         : inline-flex;
  align-items     : center;
  gap             : 0.4em;
  font-size       : 0.8rem;
  color           : var(--text-subtle);
  text-decoration : none;
  transition      : color var(--transition);
  white-space     : nowrap;
}
.app-footer__link:hover {
  color: var(--text-muted);
}
.app-footer__link svg {
  flex-shrink: 0;
}
```

Add the responsive override inside the existing `@media (max-width: 640px)` block:

```css
  .app-footer {
    flex-direction : column;
    align-items    : center;
    text-align     : center;
    gap            : 0.5rem;
  }
```

### Step 3 — No JavaScript changes, no tests

The footer is pure static markup. `refreshIcons()` in `initUI()` already
materialises any `<i data-lucide="…">` present in the DOM at load time, including
the GitHub icon in the footer.

No unit tests are required (no business logic introduced).
Coverage is unaffected.
