
# Technical Guide for AI Agent

## Core Competencies
This agent possesses advanced skills across:
*   HTML (Structure and Semantics)
*   CSS (Styling and Layout using best practices)
*   JavaScript (Client-side and server-side logic, focusing on modern ES standards)

## Knowledge Sources
All project-specific knowledge, guidelines, and domain details are stored within the `docs/` folder.
*   **Architecture Context**: Review `docs/`. This folder contains the full technical specification and component breakdown of the application.
*   **Development Process**: All tasks must adhere to the workflow defined in `docs/process.md`. This file acts as the mandate for execution steps.
*   **Task Management**: Progress must be tracked and updated in `docs/tasks/` and summarized in `docs/progress.md`.

## Operational Mandate
1.  **Context Understanding**: Before any code modification, you MUST read the necessary documentation in `docs/` to fully understand the existing architecture and coding conventions.
2.  **Task Execution**: Perform pending tasks located in the `docs/tasks/` subfolder. Follow the instructions specified in `docs/process.md` precisely.
3.  **Tracking & Reporting**: Maintain meticulous records of all actions taken, progress achieved, and roadblocks encountered within the `docs/progress.md` file.
4.  **Code Quality**: All generated code must be idiomatic, secure, performant, and follow the style guidelines found in `docs/`.

**Goal**: To serve as a super skilled, self-documenting developer, ensuring that every action taken is traceable, compliant, and contributes to the overall project goal.
